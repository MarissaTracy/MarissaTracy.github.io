May 2019

2048 - a11y

2048 is a popular tile game that was very popular back in 2014. In this game, users swipe (or use arrow keys) to move all the tiles of the board in a direction. Tiles of the same value merge together and add their values. The goal of the game is to reach a 2048 tile by merging together tiles. Users start with either a "2" or "4" tile and after every swipe (or keypress) a random "2" or "4" tile is added. However, we found that this game was not accessible for many reasons.

The color contrast in the game is not AA standards compliant. For color contrast to be AA compliant, the foreground to background color contrast must be at least 4.5:1. All of the color contrast between tile and tile and between tile and the number on the tile does not meet the standard. To make the game more accessible, we updated the colors of the board to be AA contrast compliant. We also added patterned backgrounds to the tiles so that they could be distinguished by people with monochromatic color blindness. In addition to color contrast, we found that the game is not compatible with screen readers. Because of this, the game is almost impossible to play with a screen reader because it does not allow users to know what number is in each cell. To accommodate screen readers, we updated the layout of the game board from divs to an Aria table. Now, screen readers are able to go into each row and box of the game board. QuizO

Giving in class quizzes can be difficult and the difficulty only increases as class size increases. Not only does it take time for teachers to manually grade all their students quizzes, but it leaves students in the dark about where they stand in class. QuizO is an interactive web application that allows teachers to easily offer in class quizzes to students that provides immediate feedback.

QuizO is a web application that allows teachers to give their students quiz from their web browser. QuizO uses Django and a MySQL database. There are two different types of users - a host and a student. The host can able to create quizzes that multiple students can join via a unique ID number. Each participant can optionally be aliased with a nickname of their choice. Each player will be showed their answer after each question and given a total score after the quiz is completed.

Repo Robot


Our goal was to make Sparki repossess hot wheel cars. We were able to control Sparki by using 6 different hand gestures. We wrote computer vision algorithm in Python using OpenCV that was able to count the number of fingers. The 6 gestures corresponded to the following movements: stop, move forward, turn left, turn right, close gripper, and open gripper. These 6 gestures allowed us to control Sparki to repossess cars.

To make the hand recognition algorithm, we used OpenCV in Python. After we would read in the raw image, we would "mask" the image. This turned the image into a binary version of the image so it just appeared in two colors - black or white. After that, we were able to call the built in OpenCV functions "FindContours", "convexHull" and "convexityDefects" to help us calculate how many fingers were in the image. The function that calculates the number of fingers returns an integer. This integer gets passed to our C code on Sparki as a byte over the serial. This byte corresponds to a Sparki movement.

A.I Dermatologist

Skin cancer is one of the most popular types of cancer, but it can be cured if it is caught early enough. Unfortunately, the human eye is not good at detecting skin cancer. Because of this, there are many unnecessary biopsies performed every year. By creating a neural network that learns to identify skin cancer, A.I. Dermatologist can help reduce the number of unnecessary biopsies performed each year while increasing the number of skin cancer cases caught.

A.I. Dermatologist utilizes the ISIC dataset of over 22,000 images to train a neural network to identify skin cancer. To expand the dataset, we augmented each image. Each image was augmented three ways: flipped over the y-axis, rotated 90 degrees, and rotated 180 degrees. Using OpenCV, the skin lesion was able to be identified in each image. All the images were stored in different buckets on Amazon Web Services. The use of AWS buckets allowed us to split the data up 34% training/66% testing. The images were run through a neural network that would classify a skin lesion as benign or malignant.

Whistler


In Guardians of the Galaxy, Yondu is able to control his red arrow with his mind. Whistler is an Oculus game that allows the user to step into Yondu's shoes and kill enemies with their own arrow. Each enemy randomly spawns in the scene and progressively moves closer to the user. The arrow follows the users gaze and the user can either send the arrow away or bring it closer by using the left and right trigger of the Oculus. The game was created in Unity. When the arrow collides with an enemy, a particle effect is applied to the enemy and it will explode into small cubes.

ClutterSight

Every second approximately 6,000 tweets are tweeted. Many users tweet about current events, companies, and their personal lives. ClutterSight is a web application that allows a company or person to search for a keyword and ClutterSight searches all recent tweets for that keyword. Then, ClutterSight categorizes each tweet as either "positive", "negative", or "neutral". This data is displayed via a pie chart showing the user the percentage breakdown of the tweets.

Each tweet was read in using Apache Storm. This allowed the tweets to be categorized in real-time. Each tweet was ran through a machine learning algorithm that assigned each tweet a sentiment and then stored the tweet and sentiment in a MySQL database. The tweets were pulled from the database though a PHP script and updated the pie chart as new tweets came in or when the search criteria changed.

ETHey

There is nothing more frustrating then when you pick up a friend only for them to text you "give me 10 minutes" when you finally arrive. To avoid the frustration, it is always temping to send them the "here" message when you are still a few minutes out. Although everyone thinks that they will be fine to look down at their phone for 10 seconds to send the text message, the reality is that any distraction on the road could cause an accident. ETHey eliminates the need to text and drive by automatically sending messages for users while they are driving.

ETHey utilizes your phones GPS services to track your location as you drive. You tell ETHey your destination, who you want to alert, and when. ETHey will send messages based off of your location or your time of arrival - it's your choice! ETHey also lets you set the message before you begin driving to eliminate any distractions behind the wheel. If you are picking up multiple people, ETHey can help with that too. Simply enter your route and ETHey will tell the next person when you are on the way. ETHey was developed as an iOS Application using Swift and a MySQL database.
